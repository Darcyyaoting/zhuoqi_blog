  • 不同的硬件、操作系统之间的通信需要统一成一种规则，叫做协议（protocol）

TCP/IP 协议族

  1. TCP/IP

    分层：应用层、传输层、网络层、链路层；为什么需要分层：因为需要改变时不需要改变整个协议，而是改变某层协议就行了。
    1.1 应用层
    应用层决定了向用户提供应用服务时通信的活动，如：FTP（file transfer protocol）、DNS（domain name system）、HTTP（hypertext transfer protocol）
    1.2 传输层
    传输层对上层应用层，提供处于网络连接中的两台计算机之间的数据传输，如：TCP（transmission control protocol）、UDP（user data protocol）
    1.3 网络层
    网络层用来处理在网络上流动的数据包。数据包是网络传输的最小数据单位，网络层规定了通过怎样的路径到达对方计算机，并把包给对方
    1.4 链路层
    用来处理连接网络的硬件部分
    1.5 TCP/IP 通信传输流

  2. 与http关系密切的协议：IP/TCP/DNS

    2.1 IP(internet protocol)
    IP协议的作用是把数据包传输给对方，要保证确实传输到某个地方，则需要满足某些条件，其中最重要的是IP地址和mac地址（media access control address）。
    IP地址指明了节点被分配的地址，mac地址指明了网卡所属的固定地址。ip可变，mac一般不会变。
    通常通讯双方需要经过多台计算机和网络设备才能连接到对方（局域网（LAN）除外）。IP地址间传输依靠mac地址，中转时利用下一站的mac地址搜索下一个中转目标。
    这时需要先利用ARP（address resolution protocol）通过ip反查出mac地址。ARP是解析地址的协议。

    2.2 TCP
    TCP提供可靠的字节流服务。字节流服务：为了方便传输，大块数据被分割成以报文段为单位的字节流传输。
    三次握手：发送端发出带有SYN标志的数据包给对方；接收端确认后，发回带有SYN/ACK标志的数据包；发送端再回传带有ACK的数据包，代表”握手”结束。
    如果握手期间中断，则会以同样的顺序发送相同的数据包
